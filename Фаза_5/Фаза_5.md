
# Фаза 5. Планирование

## 1. Декомпозиция проекта на задачи
Для реализации проекта «Онлайн-магазин тортов + 3D-конструктор» проведена детальная декомпозиция работ на 24 задачи. Это позволяет точно распределить ресурсы и контролировать сроки разработки сложного функционала (3D-визуализации и производственной логики).

### Перечень задач:
1.  **T01 Анализ требований:** Сбор ТЗ по 3D-моделям и расчету цен.
2.  **T02 Проектирование архитектуры:** Описание взаимодействия Front/Back.
3.  **T03 Структура БД:** Проектирование схем хранения 3D-конфигураций.
4.  **T04 Wireframes:** Прототипирование интерфейсов.
5.  **T05 UI Design:** Визуальный дизайн и подготовка ассетов.
6.  **T06 Реализация Auth API:** Авторизация и профили пользователей.
7.  **T07 API Каталога:** Управление ассортиментом готовых тортов.
8.  **T08 API Конструктора:** Сохранение уникальных параметров заказа.
9.  **T09 Логика заказов:** Обработка платежей и статусов.
10. **T10 Алгоритм выгрузки на производство:** Генерация ТЗ из 3D-данных.
11. **T11 Инициализация 3D-сцены:** Настройка Three.js движка.
12. **T12 Генератор геометрии:** Динамическое создание форм торта.
13. **T13 Модуль кастомизации:** Наложение текстур, надписей и фото.
14. **T15 Корзина и оформление:** Логика расчёта стоимости.
15. **T14 Интерфейс витрины:** Верстка каталога.
16. **T16 Админ-панель (UI):** Интерфейс управления заказами.
17. **T17 Подготовка тест-плана:** Сценарии тестирования 3D и API.
18. **T18 Функциональное тестирование:** Проверка бизнес-логики.
19. **T19 Тестирование 3D-модуля:** Проверка корректности рендеринга.
20. **T20 Интеграционное тестирование:** Проверка связей системы.
21. **T21 Исправление дефектов:** Багфикс по итогам тестов.
22. **T22 Подготовка документации:** Руководства пользователя/админа.
23. **T23 Развертывание системы:** Настройка сервера и запуск.
24. **T24 Приемка проекта:** Финальный аудит.

---

## 2. Исходные данные и ресурсы
Планирование велось с учетом ограничений по ресурсам и связей между задачами, определенных на предыдущих этапах:
*   **Состав команды:** 5 специалистов (TL, FE, FS, QA, BA), что соответствует кадровым рекомендациям из [Фазы 3](../Фаза_3/).
*   **Ресурсные требования:** Каждая задача закреплена за специалистами согласно их профилю (информация в [Фазе 2](../Фаза_2/)).
*   **Лимиты:** Рабочее время ограничено 8 часами на человека в день.

---

## 3. Решение задачи планирования
Для минимизации сроков реализации (makespan) проект был оптимизирован с помощью 12 различных эвристик упорядочивания работ.

**Инструмент расчета:**
*   [Скрипт эвристического планирования (Colab)](https://colab.research.google.com/drive/1ZN8hxeQo1LqborDx0OLJVIe0H_VipOQi?usp=sharing)

### Сравнение эвристик:
| Эвристика | Длительность (дни) |
| :--- | :--- |
| **SLK** (Общий резерв) | 82.00 |
| **LST** (Позднее начало) | 81.00 |
| **LFT** (Позднее завершение) | 81.00 |
| **LPT** (Длительность работы) | 82.00 |
| **MIS (Число последователей)** | **79.00** |
| **GRD** (Затраты ресурсов) | 82.00 |

**Лучшее решение:** Эвристика **MIS** (Most Immediate Successors) показала минимальный срок — **79 рабочих дней**. Она отдает приоритет задачам, которые "разблокируют" наибольшее количество последующих работ.

Результат Фазы 5 (**15.8 недель**) подтверждает корректность оценок, полученных ранее:
1.  Оценка методом UCP в [Фазе 3](../phase_3/) прогнозировала **15.1 неделю**.

Таким образом, выбранный темп работы и состав команды позволяют реализовать проект «Магазин тортов» в установленный срок (около 4 месяцев).
